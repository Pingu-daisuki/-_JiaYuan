JiaYuan 使用的 Vue 知识点及其项目落点
=======================================

说明
----
项目采用 Vue 3 + Vite + Element Plus，主页面全部使用单文件组件（SFC）和 Composition API。
本文除 Vue 本身外，也包含在 Vue 组件中实际使用的浏览器 API、流式请求、Markdown 渲染和工程化知识点。


一、Vue 3 应用创建与插件
-----------------------

1. createApp
   - 位置：frontend/src/main.js。
   - createApp(App) 创建应用实例，最后 app.mount('#app') 挂载到 index.html 的 #app。

2. 根组件
   - 位置：frontend/src/App.vue。
   - App.vue 负责整体布局、侧边导航和各业务页面切换，是组件树根节点。

3. app.use 插件
   - main.js 使用 app.use(ElementPlus)，让所有组件可直接使用 el-button、el-dialog、el-tree 等。

4. 全局组件注册
   - main.js 遍历 @element-plus/icons-vue 的导出并 app.component(key, component)。
   - 因此模板可直接写 <Folder/>、<Delete/> 等图标；部分业务组件也显式 import 图标，便于静态分析。

5. 全局样式导入
   - main.js 导入 assets/main.css 和 element-plus/dist/index.css。
   - ChatView 额外导入 highlight.js 与 KaTeX CSS。


二、Vue 单文件组件 SFC
---------------------

1. <template>
   - 描述组件 DOM 和 Element Plus 组件结构。
   - 所有业务页面都把列表、表单、弹窗和状态展示放在 template。

2. <script setup>
   - App.vue、ChatView.vue、LibraryPanel.vue、CampusView.vue、DeadlineBoard.vue、OjView.vue、
     SettingsView.vue 等都使用 <script setup>。
   - 导入的组件和顶层变量自动暴露给模板，不需要 export default/components/setup 返回对象。

3. <style scoped>
   - 业务组件用 scoped 把样式限制在当前组件，避免不同页面类名互相污染。
   - App.vue 同时有非 scoped 全局 reset 和 scoped 布局样式。

4. 深度选择器/全局补充
   - ChatView 对 Markdown、KaTeX、Element Plus 内部结构使用非 scoped 样式或 :deep，处理动态 v-html 内容。


三、响应式基础：ref、reactive、computed
---------------------------------------

1. ref 基本类型
   - App.vue：activeMenu 保存当前页面。
   - ChatView.vue：userInput、loading、uploadDialogVisible、pendingUrl、retrievalScopeMode 等。
   - LibraryPanel.vue：files、searchQuery、currentPath、弹窗状态。
   - SettingsView.vue：pdfEngine、activeModelName、initLogs、useGpu。
   - 访问/修改时脚本使用 .value，模板会自动解包。

2. ref 数组
   - ChatView.messages、pendingFiles、uploadLogs。
   - CampusView.savedAccounts、logs。
   - LibraryPanel.allFolders、searchResults。
   - 数组 push/splice 或整体赋值都能触发模板更新。

3. ref DOM 引用
   - ChatView.scrollbarRef、terminalBodyRef。
   - CampusView/OjView.terminalBodyRef。
   - 模板通过 ref="..." 绑定，脚本在 nextTick 后操作滚动位置。

4. reactive 对象
   - CampusView.accountForm、strategyForm。
   - OjView.accountForm。
   - 适合多个表单字段共同维护，模板可直接 v-model="accountForm.student_id"。

5. computed 派生状态
   - ChatView.canStartImport：根据模式、队列、URL、运行状态计算按钮是否可用。
   - LibraryPanel.currentFolderId：取路径最后一项。
   - LibraryPanel.visibleFolders：过滤当前父目录下文件夹。
   - LibraryPanel.isSearchMode：由 searchQuery 是否为空决定界面模式。
   - computed 避免在模板里重复复杂表达式，并按依赖缓存结果。

6. 模块级共享 ref
   - store/useProfile.js 在模块顶层 export globalProfile = ref(...)，多个导入方理论上共享同一实例。
   - 当前没有组件实际导入它，因此它是未启用的轻量状态草稿，不是 Pinia store。


四、模板语法和指令
------------------

1. 文本插值 {{ }}
   - 各页面显示文件名、状态、账号名、日志和 Deadline。
   - Vue 会自动转义普通插值，适合不可信纯文本。

2. v-if / v-else-if / v-else
   - App.vue 根据 activeMenu 选择页面。
   - ChatView 根据消息角色、sources、拖拽状态、导入模式显示不同区域。
   - LibraryPanel 根据搜索模式、空数据、文件状态显示不同内容。
   - SettingsView 根据引擎是否初始化和成功/失败显示操作按钮。

3. v-for
   - 消息、来源、文件队列、课程树、账号、模型配置、日志、Deadline 和搜索结果都使用 v-for。
   - 例：ChatView 的 messages/source，LibraryPanel 的 folders/files/results，SettingsView 的 savedConfigs。

4. :key
   - 列表通常使用数据库 id、student_id、文件 key 或组合引用 key。
   - 稳定 key 帮助 Vue 正确复用 DOM，避免列表更新错位。

5. v-model
   - el-input、el-select、el-dialog、el-radio-group、el-tree-select 与表单字段双向绑定。
   - SettingsView：pdfEngine、useGpu、新模型字段。
   - ChatView：用户输入、导入 URL、检索课程/文件范围。
   - LibraryPanel：搜索词、新文件夹、目标目录。

6. v-html
   - ChatView 使用 v-html="renderMarkdown(msg.content)" 显示 Markdown 结果。
   - 因 v-html 不转义 HTML，MarkdownIt 被明确设置 html:false，避免模型或导入文档注入原生 HTML。

7. v-else-if 组件切换
   - App.vue 没有引入 Vue Router，而是用一组 v-if/v-else-if 手动切换单页模块。

8. 属性绑定 :prop
   - :loading、:disabled、:class、:type、:data、:props、:default-expanded-keys 等把响应式值传给 DOM/组件。

9. 动态 class
   - :class="{ active: condition }"：账号选中、拖拽高亮、菜单状态。
   - :class="['urgent-text', getStatusClass(...)]"：Deadline 紧急度。

10. 事件绑定 @event
   - @click：发送消息、新建/删除/移动、选择账号、确认初始化。
   - @keydown.enter：聊天回车、新文件夹提交。
   - @click.stop：列表项内操作按钮不触发外层打开事件。
   - @click.prevent：面包屑链接不执行默认跳转。
   - @dragenter.prevent、@dragover.prevent、@dragleave.prevent、@drop.prevent：文件拖拽导入。

11. 可选链 ?. 和空值逻辑
   - 模板和脚本使用 selectedAccount?.student_id、source?.section 等，避免空对象报错。


五、组件树、组合与状态保留
--------------------------

1. 组件导入
   - App.vue 导入六个实际业务页：ChatView、LibraryPanel、CampusView、DeadlineBoard、OjView、SettingsView。
   - TheWelcome.vue 导入 WelcomeItem 和五个图标，但整个欢迎页未被 App 引用。

2. KeepAlive
   - App.vue 用 <KeepAlive> 包裹业务页面。
   - 页面切换时组件实例可被缓存，用户返回后输入和部分本地状态不会全部重建。

3. onActivated
   - OjView.vue 使用 onActivated，在 KeepAlive 缓存页再次激活时重新读取模型配置。
   - 这是 KeepAlive 特有生命周期之一。

4. 当前没有 props/emits 的主要业务组合
   - 六个主业务页直接请求后端或 localStorage，App 只控制显示，没有复杂父子 props/emits。
   - HelloWorld.vue 展示了 defineProps 示例，但它未用于实际主界面。

5. 插槽
   - 脚手架 WelcomeItem.vue/TheWelcome.vue 使用具名/默认 slot 组织欢迎内容，当前不在实际页面链路。
   - Element Plus 组件内部大量使用 template slot，如对话框 footer、下拉选项自定义内容。


六、生命周期
------------

1. onMounted
   - ChatView：加载课程/检索范围等初始数据。
   - LibraryPanel：读取课程和根目录文件。
   - CampusView：读取账号。
   - DeadlineBoard：读取账号。
   - OjView：读取账号和模型配置。
   - SettingsView：读取 localStorage，并查询当前引擎状态。

2. onBeforeUnmount
   - LibraryPanel：清除搜索 setTimeout，避免组件卸载后仍触发请求。

3. onActivated
   - OjView：页面从 KeepAlive 缓存中重新显示时刷新模型配置。

4. nextTick
   - ChatView、CampusView、OjView 在响应式日志/消息写入 DOM 后滚动到底部。
   - nextTick 确保读取 scrollHeight 时 DOM 已经完成本轮更新。


七、事件和交互设计
------------------

1. 键盘事件
   - ChatView.handleEnter 区分 Enter 发送与 Shift+Enter 换行。
   - LibraryPanel 新建文件夹输入框支持 Enter 提交。

2. 事件冒泡控制
   - 文件/文件夹卡片整体可点击，内部编辑/删除按钮使用 @click.stop，防止误打开。

3. 拖拽文件
   - ChatView 在输入区域监听 dragenter/dragover/dragleave/drop。
   - drop 后从 event.dataTransfer.files 取得 FileList，进入统一 addPendingFiles 校验与队列。

4. 多文件队列
   - pendingFiles 保存 key、File、documentType、status、error。
   - executeImport 使用 for...of 顺序上传，避免 Marker/MinerU 多文件同时抢 GPU。

5. 防重复提交
   - loading、isUploading、isRunning、isSavingAccount 等 ref 控制按钮 disabled/loading。

6. 确认框和输入框
   - Element Plus ElMessageBox.confirm：删除模型、文件夹、文件、账号。
   - ElMessageBox.prompt：重命名文件夹。

7. 即时消息
   - ElMessage.success/warning/error 给出短反馈。


八、HTTP 请求与异步 JavaScript
------------------------------

1. Fetch API
   - ChatView、LibraryPanel、CampusView、OjView、SettingsView 使用 fetch。
   - 明确设置 method、headers、JSON.stringify(body) 或 FormData。

2. Axios
   - DeadlineBoard 使用 axios 获取账号、Deadline 和触发同步。
   - 展示项目同时兼容 fetch 和 axios 两种风格。

3. async / await
   - 前端所有网络流程使用 async/await，使顺序、异常和 loading 状态更清晰。

4. try / catch / finally
   - 请求失败在 catch 中显示 ElMessage 或写终端日志。
   - finally 重置 loading/isUploading/isSaving 等状态，避免按钮永久锁死。

5. HTTP 状态检查
   - 上传、引擎初始化和检索在读取正文前检查 response.ok。
   - ChatView.parseHttpError 尝试读取后端 detail，给用户真实错误而非只显示 statusText。

6. FormData
   - ChatView.uploadOneDocument 把 file、course_id、engine 组成 multipart/form-data。
   - 浏览器自动生成 boundary，不手工设置 Content-Type。

7. JSON 请求
   - URL 导入、检索、模型聊天、文件移动和账号策略使用 application/json。

8. ReadableStream
   - ChatView.readUploadResponse：读取文档入库日志。
   - ChatView.sendMessage：读取大模型 token 流。
   - SettingsView.runEngineInit：读取 pip/模型/探针日志。
   - OjView/CampusView 也读取流式后端输出。

9. getReader + TextDecoder
   - response.body.getReader() 循环 read() 得到 Uint8Array。
   - TextDecoder('utf-8').decode(value, {stream:true}) 正确处理中文字符跨网络块被拆开的情况。

10. 流式缓冲与分行
   - 上传/初始化日志保留未完成行 buffer，遇到换行才追加到数组。
   - 结束后检查后端成功/失败 token，而不是仅根据 HTTP 200 弹成功。

11. 请求串联
   - ChatView.sendMessage 先 /rag/retrieve。
   - 无依据直接显示检索结果，不调用模型。
   - 有依据再 /chat/，把 context 和模型配置一起发送。


九、localStorage 与客户端持久化
-----------------------------

1. 模型设置
   - SettingsView 把 rag_full_settings 序列化为 JSON 保存。
   - 内容包含当前文档引擎、激活模型和多个模型的 name/type/modelId/baseUrl/apiKey。

2. 跨页面读取
   - ChatView 读取 rag_full_settings 决定解析引擎和聊天模型。
   - OjView 读取同一配置提供可用模型。

3. 资料树缓存
   - LibraryPanel 把 rag_library_tree 写入 localStorage。
   - ChatView 导入对话框读取它快速生成目标课程树，同时仍从后端刷新真实目录。

4. 序列化
   - JSON.stringify 写入，JSON.parse 读取；读取放在 try/catch 或默认值保护中。

5. 安全边界
   - localStorage 可被同一 Electron profile/页面脚本读取，API Key 不是加密保存。
   - 当前适合单用户本地工具；若多人共享设备，应改用主进程安全存储或 Windows Credential Manager。


十、文件导入和浏览器 File API
----------------------------

1. Element Plus Upload
   - ChatView 使用 drag、multiple 和 accept 属性提供点击/拖拽文件入口。
   - 正式上传由自定义逻辑接管，不依赖组件自动 action。

2. File / FileList
   - 从 el-upload 回调或 dataTransfer.files 获得 File。
   - 使用 file.name、file.size、file.type、lastModified 生成队列 key 和校验信息。

3. 前端格式白名单
   - SUPPORTED_DOCUMENT_TYPES 支持 PDF、DOCX、PPTX、MD/MARKDOWN、HTML/HTM。
   - MAX_DOCUMENT_BYTES=100 MiB，提前阻止明显错误；后端仍会独立严格校验。

4. 队列状态
   - queued、processing、ready、failed 映射为中文文字和 Element Plus tag 类型。

5. URL 导入
   - importMode 在 files/url 之间切换。
   - 网页安全检查由后端完成，前端只提供输入、状态和错误展示。


十一、Markdown、代码和公式渲染
------------------------------

1. markdown-it
   - ChatView 创建 MarkdownIt 实例，把模型文本转换成 HTML。
   - html:false：禁止输入中的原生 HTML。
   - linkify/breaks 等配置改善回答显示。

2. highlight.js
   - MarkdownIt highlight 回调根据语言调用 hljs.highlight。
   - 未知语言使用转义后的普通 code block，避免抛错。

3. markdown-it-texmath + KaTeX
   - md.use(texmath, {engine:katex,...}) 支持数学公式。
   - throwOnError:false 让不完整公式不致使整个回答渲染失败。

4. v-html 安全
   - 最终 HTML 通过 v-html 插入。
   - 当前主要防线是 html:false 与高亮转义；如果未来允许用户自定义 HTML，仍建议增加 DOMPurify。


十二、引用来源和多格式打开
--------------------------

1. sources 响应式数据
   - assistant 消息对象同时保存 content 和 sources。
   - 模板只在 assistant 且 sources 非空时渲染引用 chip。

2. 位置标签
   - sourceLocationLabel 根据 location_type 显示“第 N 页”“第 N 张幻灯片”“章节：...”或网页章节。

3. PDF
   - openSource 为 PDF content URL 添加 #page=N，由浏览器 PDF 查看器定位。

4. DOCX/PPTX
   - 通过后端 content 路由打开/下载，由操作系统 Office 应用处理。

5. Markdown/HTML
   - 调用 preview API，在 Element Plus Dialog 中以纯文本形式预览。

6. URL 来源
   - window.open(source_url, '_blank', 'noopener,noreferrer') 打开原网页并隔离 opener。


十三、资料库 UI 技术点
----------------------

1. 树形数据转换
   - LibraryPanel.buildTree 把带 id/parent_id 的平面课程数组转为 children 树。
   - ChatView 也构建检索课程树和上传目标树。

2. 面包屑路径
   - currentPath 是从根到当前目录的数组。
   - navigateTo(index) 截断数组实现返回上层。
   - buildPathToFolder 根据 parent_id 反向组装搜索结果所在路径。

3. 搜索防抖
   - searchQuery 输入后 scheduleSearch 清除旧 timer，再 setTimeout(searchLibrary, 250)。
   - onBeforeUnmount 清 timer，避免卸载后更新状态。

4. 搜索模式
   - isSearchMode computed 决定显示目录内容还是全库搜索结果。

5. 状态展示
   - STATUS_LABELS/statusTagType 把 uploaded/parsing/indexing/ready/failed 映射为用户文本和颜色。
   - ready 时显示页/幻灯片/章节数、chunk 数、引擎和耗时。


十四、设置页和引擎初始化 UI
--------------------------

1. 受控引擎选择
   - pdfEngine 是当前选择，previousEngine 用于取消或失败时回滚。

2. 二阶段确认
   - 选 Marker/MinerU 后先查询 status。
   - 未初始化时打开 CPU/GPU 选择弹窗；用户确认后才开始下载。

3. 流式状态机
   - initDone、initFailed、initLogs 控制弹窗内容和按钮。
   - 只有读取到后端 INIT_SUCCESS_TOKEN 且再次 status 校验 initialized=true 才提示成功。

4. 配置 CRUD
   - savedConfigs 保存模型数组。
   - saveNewModel 检查必填和重名。
   - confirmDeleteModel 使用 ElMessageBox。
   - activeModelName 指向当前使用模型。

5. 敏感输入
   - API Key 使用 type="password" 和 show-password，仅改变显示方式，不等于加密存储。


十五、校园、Deadline 和 OJ 页面
------------------------------

1. CampusView
   - reactive 表单管理账号与复杂策略。
   - time_slots 数组动态 push/splice，实现增删时间段。
   - 流式日志根据关键字映射颜色，并 nextTick 自动滚动。

2. DeadlineBoard
   - 用 axios 获取数据。
   - 时间函数将 deadline 与当前时间比较，返回状态文本、Timeline 类型和 CSS class。
   - Element Plus Timeline 展示按时间排序的任务。

3. OjView
   - 组合账号、实验、语言、模型和间隔参数。
   - onActivated 刷新 localStorage 模型列表。
   - 读取 text/event-stream，按日志类型着色，并支持停止前端读取控制。


十六、Element Plus 的具体使用
----------------------------

1. 布局：el-container、el-aside、el-main。
2. 导航：el-menu、el-menu-item。
3. 表单：el-input、el-select、el-option、el-radio-group、el-form、el-form-item。
4. 文件：el-upload、el-upload-dragger。
5. 树：el-tree-select 或树形选项，用于课程目标和检索范围。
6. 弹窗：el-dialog。
7. 按钮/状态：el-button、el-tag、el-empty、loading/disabled。
8. 反馈：ElMessage、ElMessageBox.confirm/prompt。
9. 图标：@element-plus/icons-vue 与 el-icon。
10. 时间线：el-timeline、el-timeline-item。
11. 滚动：el-scrollbar 及组件 ref。


十七、CSS 与布局知识点
--------------------

1. Flexbox
   - 主侧栏、工具栏、消息气泡、文件卡片、模型卡片普遍使用 display:flex。

2. Grid
   - 资料卡片、设置区域按需要采用 grid 布局。

3. 高度链
   - html/body/#app、App 容器、el-container 设置 100%，保证桌面窗口内完整铺满。

4. overflow
   - 页面外层 hidden，内部消息/列表区独立滚动，避免整个 Electron 窗口双滚动条。

5. scoped 样式
   - Vue 编译器为元素附加 data 属性，使选择器局部化。

6. 动画
   - App.module-wrapper 使用 fadeIn keyframes，让模块切换更平滑。

7. 状态色
   - 上传、入库、日志、Deadline 等通过 class/tag type 区分成功、警告、错误和处理中。


十八、Vite 与前端工程化
----------------------

1. Vite 开发和构建
   - npm run dev：开发服务器和 HMR。
   - npm run build：生产静态资源输出到 dist。
   - npm run preview：本地预览 dist。

2. Vue 插件
   - @vitejs/plugin-vue 负责编译 .vue SFC。

3. 相对 base
   - vite.config.js 设置 base:'./'，让生产资源在 Electron file:// 下正确加载。

4. 开发工具条件加载
   - mode==='development' 才启用 vite-plugin-vue-devtools，生产包不带开发工具。

5. 路径别名
   - @ 指向 frontend/src；jsconfig.json 同步告诉编辑器这一映射。

6. ESLint
   - eslint.config.js 使用 flat config、JS recommended 和 Vue essential。

7. Oxlint
   - .oxlintrc.json 开启 correctness 错误规则；npm 脚本与 ESLint 串行执行。

8. Prettier
   - .prettierrc.json 定义无分号、单引号和 100 列。

9. package-lock
   - 固定完整依赖版本，CI/其他开发机可用 npm ci 得到一致依赖树。

10. Node 版本
   - package.json engines 要求 Node ^22.18.0 或 >=24.12.0。


十九、Electron 环境下的前端适配
----------------------------

1. file:// 静态加载
   - Vite 相对 base 让 index.html 可以从本地文件系统加载资源。

2. API 端口重定向
   - Vue 组件当前请求 http://127.0.0.1:8000。
   - desktop/main.cjs 的 webRequest 把它透明改到桌面后端 8765。

3. 外链
   - window.open 由 Electron setWindowOpenHandler 拦截后交给系统浏览器，不在应用窗口加载不可信网站。

4. localStorage 隔离
   - Electron userData 指向 %LOCALAPPDATA%\JiaYuan\profile，设置不会写到安装目录。

5. 浏览器安全边界
   - BrowserWindow 开启 contextIsolation、sandbox、webSecurity，关闭 nodeIntegration。
   - 因此 Vue 页面无法直接 require Node 模块，仍按普通浏览器前端编写。


二十、当前 Vue 实现中没有使用或可改进的点
----------------------------------------

1. 没有 Vue Router
   - 当前用 activeMenu + v-if 切页，规模继续增长时可引入 Router、URL 状态和懒加载。

2. 没有 Pinia
   - localStorage 和各组件自行取数；模型配置、账号摘要、课程树可抽成 Pinia store/composable。

3. API 地址重复
   - 多个组件硬编码 127.0.0.1:8000。建议抽成 src/api/client.js 或 VITE_API_BASE。

4. fetch 与 axios 混用
   - DeadlineBoard 使用 axios，其余主要用 fetch。可统一客户端、错误格式和超时策略。

5. 未使用脚手架组件
   - HelloWorld、TheWelcome、WelcomeItem、icons 和 logo.svg 未进入主 App，可删除以降低认知负担。

6. useProfile.js 未使用
   - 如果不计划做全局账号状态，可删除；若要使用，建议改成真正 composable 或 Pinia store。

7. localStorage 敏感数据
   - API Key 当前明文。桌面版可通过 preload + IPC 调用系统凭据存储。

8. v-html 防护
   - html:false 已降低风险，但第三方 Markdown 插件或未来配置变化仍可能引入 HTML；建议增加 DOMPurify。

9. 组件体积
   - ChatView.vue 同时承担导入、检索、消息、Markdown、引用和预览，可拆成 ImportDialog、MessageList、
     RetrievalScope、SourceList 等子组件和 composable。

10. 自动测试
   - 当前没有 Vitest/Vue Test Utils 测试。建议覆盖流式 UTF-8 分块、成功 token、文件队列、引用定位、
     搜索防抖、引擎失败回滚和 localStorage 迁移。
