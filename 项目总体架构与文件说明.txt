JiaYuan 项目总体架构、搭建思路、技术栈与文件说明
====================================================

文档说明
--------
本文按当前工作区代码编写，描述的是实际实现，不是理想化设计稿。
“每个文件”以项目自有源码、配置、脚本和交付物为范围。node_modules、Python 私有运行时、
Tesseract 安装目录等第三方内容包含数万文件，本文按目录和关键文件类型整体说明，不逐个解释第三方源码。


一、项目定位
------------
JiaYuan 是一个 Windows 本地优先的校园工具与课程知识库应用，主要包含五组能力：

1. 课程资料 RAG：导入 PDF、DOCX、PPTX、Markdown、HTML 或网页，切片后写入 Chroma，
   按课程/文件范围检索，并把页码、幻灯片或章节作为引用返回。
2. 文档解析：默认使用 PyMuPDF；扫描 PDF 使用 Tesseract OCR 兜底；复杂版面可选 Marker/MinerU。
3. 资料库管理：课程树、文件移动/删除、入库状态、模糊搜索、失败重建和来源预览。
4. 校园工具：统一身份认证、签到策略、课程作业/考试截止时间同步。
5. OJ 工具：读取实验题目，调用大模型生成代码，提交并轮询评测结果。

应用既可以采用“Vue 开发服务器 + FastAPI”方式开发，也可以通过 Electron 打包为一个 Windows
一键安装程序。桌面版把 Python、后端依赖、Tesseract 和基础向量模型一并携带，不要求用户预装开发环境。


二、总体分层架构
----------------

  [Vue 3 + Element Plus 前端]
              |
              | HTTP / 流式 fetch
              v
  [FastAPI 路由层 backend/routes]
              |
              v
  [业务核心层 backend/core]
      |          |             |
      |          |             +-- 校园认证 / OJ / 外部大模型
      |          +-- PyMuPDF / Tesseract / Marker / MinerU
      +-- 文档校验、切片、混合检索、状态补偿

  Windows 桌面模式：
  [NSIS 安装器] -> [Electron 主进程] -> 解压私有 Python/模型 -> 启动 FastAPI:8765
                                      -> 加载 Vue 静态页面
                                      -> 把页面中的 8000 请求重定向到 8765


三、核心业务流程
----------------

1. 文档入库流程

   前端拖拽/选择文件
      -> POST /api/rag/upload
      -> 扩展名、MIME、文件签名、大小和 Office ZIP 安全校验
      -> 计算 SHA-256，检查是否已有 ready 记录可以复用
      -> 创建 knowledge_files 记录，状态 uploaded
      -> parsing：按格式/引擎提取结构化文本
      -> indexing：按章节和重叠窗口切片
      -> 写入 Chroma，并保存 file_id、course_id、页码、章节、chunk_index、engine 等元数据
      -> ready：记录页数/单元数、耗时和 chunk 数
      -> 任何阶段失败：状态设为 failed，并按 file_id 补偿删除向量

2. RAG 问答流程

   用户问题
      -> POST /api/rag/retrieve，携带 course_id 和可选 file_ids
      -> Chroma 向量 Top 10
      -> BM25 关键词分数 + 查询覆盖率
      -> MMR 去重
      -> 可选 CrossEncoder reranker
      -> 最低相关度门槛
      -> 最终 Top 3 上下文与 sources
      -> 若无充分依据，直接返回“资料库没有足够依据”
      -> 若有依据，再 POST /api/chat/ 调用 OpenAI 兼容接口流式生成答案
      -> 前端展示答案及可点击引用

3. Marker/MinerU 初始化流程

   设置页选择引擎与 CPU/GPU
      -> GET /api/engine/init/{engine}?use_gpu=...
      -> 检查 CLI/模块
      -> 在当前私有 Python 中 pip 安装 marker-pdf[full] 或 mineru[pipeline]
      -> GPU 模式先验证 CUDA kernel，必要时安装匹配的 PyTorch CUDA wheel
      -> MinerU 检查并下载完整模型目录；Marker 首次探针时下载 Hugging Face 模型
      -> 生成最小 PDF 并执行真实解析探针
      -> 仅探针成功后写入带版本、设备和模型目录的初始化标记
      -> 全过程以流式日志和心跳反馈给前端，并有超时和子进程回收

4. 桌面启动流程

   用户双击 JiaYuan
      -> Electron 单实例锁
      -> 显示 splash.html
      -> 比较 runtime-manifest.json
      -> 首次启动解压 python-runtime.tar 和 models-runtime.tar
      -> 以 JIAYUAN_DATA_DIR/JIAYUAN_MODEL_DIR 指向 LocalAppData
      -> 隐藏启动 backend/main.py，监听 127.0.0.1:8765
      -> 轮询 /api/health
      -> 后端就绪后加载 frontend/index.html
      -> 退出 Electron 时终止其创建的后端进程树


四、搭建思路
------------

1. 本地优先
   文档、向量和账号配置保存在用户本机，RAG 检索无需把原始资料上传到远端。只有最终大模型问答、
   网页导入或校园/OJ 接口需要网络。

2. API 与界面分离
   Vue 只负责交互和展示，FastAPI 负责文档安全、数据库、模型调用和长任务。两者通过清晰的
   /api 路由通信，便于将来替换 UI 或单独测试后端。

3. 长任务全部可观察
   文档解析、引擎安装、签到和 OJ 托管均可能耗时，因此后端使用 StreamingResponse，前端用
   ReadableStream 逐块读取日志，避免“HTTP 200 但内部早已失败”却仍提示成功。

4. 可恢复入库
   数据库状态机负责记录每一步；应用启动时会恢复异常中断记录；向量写入采用 file_id 幂等删除再写入，
   失败时补偿删除，从而避免数据库和 Chroma 不一致。

5. 通用文档协议
   不同解析器最终统一输出 text、section、location_type、location_index 等字段，使 PDF 页、PPT 幻灯片、
   Word/Markdown 标题和网页章节都能进入同一切片与引用流程。

6. 按需引擎
   普通文档走轻量原生解析。Marker/MinerU 体积大、依赖显卡环境，因此不写死在基础 requirements，
   而是在用户选择后由后端安装、下载、验证，并把模型放到独立用户目录。

7. 可复制桌面运行时
   Electron 不依赖系统 Python。构建脚本下载官方嵌入式 Python，安装后端 requirements，加入基础 BGE，
   验证离线导入后打成 tar，由 NSIS 与应用一起发布。


五、技术栈
----------

前端：
- Vue 3，Composition API，<script setup>。
- Vite 8：开发服务器和生产构建。
- Element Plus：布局、菜单、表单、弹窗、树、时间线、消息提示和图标。
- Fetch API / Axios：普通请求及流式响应读取。
- markdown-it：答案 Markdown 渲染。
- highlight.js：代码高亮。
- KaTeX + markdown-it-texmath：数学公式渲染。
- localStorage：模型配置、解析引擎和资料树缓存。
- ESLint、Oxlint、Prettier：静态检查和格式化。

后端：
- Python 3.13 桌面运行时；源码兼容常规 Python 3.10+ 开发环境。
- FastAPI + Uvicorn + Pydantic + python-multipart。
- SQLite：课程、文件状态、账号和截止时间。
- ChromaDB：持久化向量库及元数据过滤。
- sentence-transformers：BGE embedding 与可选 CrossEncoder reranker。
- PyMuPDF(fitz)：PDF 文本/页数/图像渲染。
- Tesseract：扫描页 OCR 兜底。
- python-docx、python-pptx、BeautifulSoup4：DOCX、PPTX、HTML 解析。
- Marker、MinerU：可选复杂版面视觉解析。
- httpx、requests、urllib3：网页抓取和校园/OJ HTTP 通信。
- OpenAI Python SDK：调用 OpenAI 兼容大模型接口。
- xmulogin：厦大统一身份认证。

桌面与构建：
- Electron：Windows 桌面壳、后端进程管理、路径隔离和外链控制。
- electron-builder + NSIS：x64 一键安装、桌面/开始菜单快捷方式。
- PowerShell：构建私有 Python 运行时与模型归档。
- npm package-lock：固定前端和桌面 Node 依赖版本。


六、API 模块划分
----------------

- GET  /api/health：桌面启动探活。
- POST /api/chat/：携带 RAG context 调用大模型并流式返回答案。
- POST /api/rag/upload：上传并入库文档。
- POST /api/rag/import-url：安全抓取网页并入库。
- POST /api/rag/files/{file_id}/reindex：失败或已有文件重新索引。
- GET  /api/rag/files/{file_id}/status：查询入库状态和指标。
- POST /api/rag/retrieve：按课程/文件范围混合检索。
- GET  /api/engine/status/{engine}：Marker/MinerU 初始化状态。
- GET  /api/engine/init/{engine}：流式执行引擎初始化。
- /api/library/...：课程树、模糊搜索、文件移动/删除、内容下载和文本预览。
- /api/campus/...：校园账号、策略和签到流。
- GET /api/deadlines/{student_id}：读取缓存或同步截止时间。
- GET /api/oj/stream_solve：OJ 登录、生成、提交、轮询的 SSE 流程。

（二）backend：FastAPI 后端

backend/main.py
  后端入口。建立 FastAPI、CORS、健康检查，挂载所有路由；启动时恢复中断入库并补齐旧文件指标；
  从 JIAYUAN_PORT 读取端口，开发模式可热重载。

backend/requirements.txt
  基础后端依赖清单。Marker/MinerU 不在这里固定安装，而由设置页按需初始化。

backend/core/__init__.py
  把 core 标记为 Python 包，便于 routes 使用 core.xxx 导入。

backend/core/paths.py
  统一资源路径和可写数据路径。开发模式默认写 backend；桌面模式通过环境变量写 LocalAppData。
  定义数据库、uploads、vector_db、engine_flags、engine_config、models 和 MinerU 配置位置。

backend/core/db.py
  SQLite 建表、迁移和连接管理。创建 courses、knowledge_files、deadlines；维护文档状态机、SHA-256、
  指标字段和索引；提供账号与 deadline 的基础读写方法。

backend/core/document_parser.py
  通用文档安全校验和轻量解析器。识别扩展名/MIME/文件签名；防 Office ZIP 炸弹和路径穿越；
  解析 DOCX 段落/表格、PPTX 幻灯片、Markdown 标题和 HTML 正文；生成统一 segment 与预览文本。

backend/core/processor.py
  RAG 核心。负责 PyMuPDF 提取、Tesseract OCR、Marker/MinerU 调用、分节切片、Chroma 写入、
  入库状态和失败补偿；负责课程/文件 where 过滤、向量候选、BM25、MMR、reranker、阈值、来源组装。

backend/core/engine_init.py
  Marker/MinerU 安装与验证中心。寻找 CLI、构造命令、管理 pip/模型下载、检查 CUDA 架构、升级 GPU
  PyTorch、执行最小 PDF 探针、流式心跳、超时终止、完整性判断和初始化标记。

backend/core/campus_core.py
  XmuNativeBot 业务类。通过 xmulogin 建立会话，判断策略时间段，抓取 deadline，发送数字码/雷达签到；
  包含坐标和双点距离求解的数学处理。

backend/core/campus_task.py
  一个异步启动本机 xmu CLI 的辅助模块，使用 asyncio 子进程并按 GBK 解码控制台。目前没有被主路由引用，
  且包含开发机绝对路径，属于遗留实现；正式校园流程使用 campus_core.py。

backend/core/oj_core.py
  OJ 核心客户端和模型网关。构造解题提示词；支持 Gemini 风格与 OpenAI 兼容调用；实现 OJ 登录、
  进入实验、读取题目、提交代码、解析限频提示和轮询评测结果。

backend/routes/__init__.py
  把 routes 标记为 Python 包。

backend/routes/chat.py
  大模型聊天路由。Pydantic 校验模型配置和请求；没有合格 RAG context 时拒绝自由猜测；有依据时通过
  OpenAI SDK stream=True 调用模型，并把增量文本作为 StreamingResponse 返回。

backend/routes/rag.py
  文档入口和检索 API。处理上传暂存、URL 导入、SHA 去重复用、数据库登记、重新索引、状态查询和范围检索；
  网页导入包含 DNS/IP/重定向/类型/大小检查以降低 SSRF 风险。

backend/routes/library.py
  资料库 API。管理树形课程、文件列表、移动/删除和向量元数据同步；提供中文友好的模糊搜索、
  文件安全下载和 Markdown/HTML 文本预览。

backend/routes/engine.py
  Marker/MinerU 状态及初始化的薄路由，把 engine_init.py 的生成器封装为流式 HTTP 响应。

backend/routes/campus.py
  校园账号与签到路由。创建 campus_config 表，管理账号和策略，通过流式日志运行 XmuNativeBot。
  当前账号密码保存在本机 SQLite，未做额外应用层加密。

backend/routes/deadlines.py
  截止时间 API。默认读 SQLite 缓存；sync=true 时读取账号、统一认证、在线抓取并刷新 deadlines 表。

backend/routes/oj.py
  OJ 托管路由。读取账号与模型配置，用 asyncio.to_thread 包装同步 HTTP 调用，通过 SSE 顺序反馈登录、
  题目、生成、提交、评测、冷却和错误日志。

backend/source/tesseract/
  随项目携带的 Windows Tesseract 运行目录（约 117 个文件）。tesseract.exe 是 OCR 主程序；相关 DLL 是
  运行库；tessdata/chi_sim.traineddata 是简体中文识别数据；configs/、tessconfigs/ 和训练工具属于
  Tesseract 发行内容。processor.py 通过 resource_path 定位此目录。

backend/campus_assistant.db、uploads/、vector_db/、engine_flags/、engine_config/、models/
  均为运行时生成目录/文件，不应提交。当前初始化状态下可以不存在；应用启动后按需创建。

backend/__pycache__/、backend/core/__pycache__/、backend/routes/__pycache__/ 及其中 *.pyc
  Python 导入模块时自动生成的字节码缓存，不是源码；不同 Python 版本会生成不同文件名。它们已被
  .gitignore 忽略，可以安全删除，之后运行后端还会自动重建。

（四）desktop：Windows 桌面封装

desktop/main.cjs
  Electron 主进程。创建安全 BrowserWindow；限制导航和外链；管理单实例；显示启动页；解压私有 Python
  与基础模型；启动/探活/关闭 FastAPI；设置 LocalAppData 数据路径；记录 electron/backend 日志；
  把前端固定的 8000 API 请求重定向到桌面后端 8765。

desktop/splash.html
  后端和模型加载期间的启动页，main.cjs 动态更新状态文字。

desktop/package.json
  Electron/electron-builder 配置。声明 appId、产品名、asar、extraResources、NSIS x64 一键安装、
  当前用户范围、桌面/开始菜单快捷方式、安装后启动和输出文件名。

desktop/package-lock.json
  固定 Electron 和 electron-builder 的完整 Node 依赖树。

desktop/scripts/build-python-runtime.ps1
  构建桌面私有运行时：下载嵌入式 Python 与 get-pip，安装后端 requirements，下载 BGE 模型，

desktop/node_modules/
  Electron 构建依赖，不提交 Git。


八、运行时数据位置
------------------

开发模式默认位于 backend：
- backend/campus_assistant.db
- backend/uploads/
- backend/vector_db/
- backend/engine_flags/

桌面模式默认位于 %LOCALAPPDATA%\JiaYuan：

九、当前实现中值得注意的边界
----------------------------

1. campus_task.py 含开发机绝对路径且当前未被调用；如重新启用应改成 PATH/配置驱动。
2. campus_config 中的账号密码和 SettingsView localStorage 中的 API Key 是本机明文数据，适合个人本地工具，
   不适合共享 Windows 账号或多人服务器部署。
3. frontend 的 API 地址仍写为 127.0.0.1:8000；Electron 通过 webRequest 重定向到 8765。长期可改成统一
   API_BASE 配置，减少桌面适配层。
4. HelloWorld/TheWelcome/WelcomeItem/icons 和 useProfile.js 是未使用脚手架/草稿，可在后续整理中删除。
5. OJ 模块关闭了部分 HTTPS 证书警告，部署到不可信网络前应恢复严格证书验证。
6. 安装程序当前未配置商业代码签名和自定义 Windows 图标。
7. 当前仓库没有系统化 pytest/Vitest 测试目录，主要依靠构建、语法检查和桌面启动烟雾测试。
